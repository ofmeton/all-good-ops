# 民泊清掃管理アプリ 検収シナリオ

このドキュメントは、アプリが期待通り動くかを確認していただくための **検収用テストシナリオ** です。順番に試して、各ステップで「期待結果」が表示されたら ✓ を付けてください。最後まで一通り通れば検収完了となります。

所要時間: 60〜90 分（通知の到着待ち含む）

---

## 0. 準備するもの

- **PC（管理者操作用）** — ブラウザは Google Chrome / Edge / Safari いずれか
- **スマートフォン**（テスト用スタッフ・オーナーになる人の端末）
  - LINE アプリがインストール済み
  - メールが受信できる
- **テスト用メールアドレス 3 つ**（管理者・テストスタッフ・テストオーナー用。同じアドレスを使い回しても OK）
- **本アプリ URL**: `https://minpaku-cleaning.vercel.app`
- **管理者ログイン情報**（工藤陸から別経路で受領）
- **LINE 公式アカウントの友だち追加 QR / URL**（工藤陸から別経路で受領。テストスタッフ役の人が事前に友だち追加しておく）

---

## 1. シナリオ A: 管理者として一周（PC で操作）

### A-1. ログイン

| 手順 | 期待結果 | ✓ |
|---|---|---|
| `https://minpaku-cleaning.vercel.app/admin/login` を開く | ログイン画面が表示される | ☐ |
| 受領した管理者メール・パスワードを入力して「ログイン」 | `/admin` ダッシュボードに遷移する | ☐ |

### A-2. 物件を登録

| 手順 | 期待結果 | ✓ |
|---|---|---|
| 左メニュー「物件管理」→「新規物件作成」 | 物件入力フォームが表示される | ☐ |
| 名称「テスト物件」、住所「東京都〇〇区テスト1-2-3」、定員「4」を入力して保存 | 物件一覧に「テスト物件」が表示される | ☐ |

### A-3. オーナーを登録 + トークン発行

| 手順 | 期待結果 | ✓ |
|---|---|---|
| 「オーナー管理」→「新規オーナー作成」 | オーナー入力フォームが表示される | ☐ |
| 名前「テストオーナー」、メール（テストオーナー用）、を入力して保存 | オーナー一覧に「テストオーナー」が表示される | ☐ |
| 物件管理 → テスト物件を開く → オーナーを割当（テストオーナーを選択） | 割当成功表示 | ☐ |
| 同物件ページで「オーナートークン発行」 | トークン URL（`https://minpaku-cleaning.vercel.app/owner/...`）が表示される | ☐ |

### A-4. スタッフを登録 + トークン発行

| 手順 | 期待結果 | ✓ |
|---|---|---|
| 「スタッフ管理」→「新規スタッフ作成」 | スタッフ入力フォームが表示される | ☐ |
| 名前「テストスタッフ」、メール（テストスタッフ用）を入力して保存（line_user_id はまだ空でOK） | スタッフ一覧に「テストスタッフ」が表示される | ☐ |
| テストスタッフを物件に担当割当 | 割当成功表示 | ☐ |
| 同スタッフページで「トークン発行」 | トークン URL（`https://minpaku-cleaning.vercel.app/staff/...`）が表示される | ☐ |

### A-5. テストスタッフの LINE userId を取得して登録

スタッフが LINE 公式アカウントを友だち追加した状態で、こちらの操作が必要です。

| 手順 | 期待結果 | ✓ |
|---|---|---|
| LINE Official Account Manager の「ユーザー」一覧を開く | テストスタッフのアイコンが表示される | ☐ |
| そのユーザーをタップして詳細を表示し、user ID（`U` で始まる文字列）を取得 | user ID が確認できる | ☐ |
| アプリの「スタッフ管理」→ テストスタッフ → 「LINE user ID」欄に貼り付けて保存 | 保存成功表示 | ☐ |

> LINE Official Account Manager の操作方法は `01-admin-guide.md` の §11「困った時」を参照。

### A-6. 依頼を作成

| 手順 | 期待結果 | ✓ |
|---|---|---|
| 「依頼管理」→「新規依頼作成」 | 依頼入力フォームが表示される | ☐ |
| 物件「テスト物件」、チェックイン日（明日以降）、チェックアウト日（チェックイン翌日以降）、人数「2」、備考「検収テスト用」を入力 | 入力できる | ☐ |
| 当日を選んで保存しようとする | 「チェックイン日は翌日以降にしてください」とエラー表示される | ☐ |
| 正しい日付に戻して保存 | 依頼一覧に「テスト物件」依頼が「未割当」で表示される | ☐ |

### A-7. （依頼作成時の通知が走ったことを確認）

→ シナリオ D で確認。

## 2. シナリオ B: スタッフとして一周（スマートフォンで操作）

### B-1. トークン URL を開く

| 手順 | 期待結果 | ✓ |
|---|---|---|
| スタッフトークン URL をスマホで開く | スタッフ用画面が表示される | ☐ |
| 画面下に「未割当の依頼」セクションがあり、シナリオ A-6 で作成した依頼が表示されている | 依頼カードが見える | ☐ |

### B-2. 依頼を承認

| 手順 | 期待結果 | ✓ |
|---|---|---|
| 依頼カードの「この依頼を承認する」をタップ | 「割当済み」状態に変わる | ☐ |

### B-3. 清掃開始

| 手順 | 期待結果 | ✓ |
|---|---|---|
| 「割当済み」依頼を開いて「清掃を開始する」をタップ | 「作業中」状態に変わる | ☐ |

### B-4. 完了報告を提出

| 手順 | 期待結果 | ✓ |
|---|---|---|
| 同画面で完了報告フォームが表示される | チェックリスト各項目とコメント欄が見える | ☐ |
| チェックリストに全部チェック、写真をスマホから2枚アップロード、コメント「テスト報告」を入力 | アップロード成功表示・写真サムネが見える | ☐ |
| 「提出する」をタップ | 「完了報告済み」状態に変わる | ☐ |

### B-5. 備品補充を依頼

| 手順 | 期待結果 | ✓ |
|---|---|---|
| 同画面下の「備品補充を依頼する」セクションに「テスト用トイレットペーパー × 1」を入力して送信 | 「依頼済み」表示 | ☐ |

## 3. シナリオ C: 管理者として完了確認（PC で操作）

| 手順 | 期待結果 | ✓ |
|---|---|---|
| 「依頼管理」→ テスト物件の依頼を開く | 「完了報告済み」状態でスタッフが提出した内容（チェックリスト・写真・コメント）が表示される | ☐ |
| 写真サムネをタップして拡大 | 写真が大きく表示される | ☐ |
| 「内容を確認済みにする」をタップ | 「確認済み」状態に変わる | ☐ |
| 「備品補充管理」を開く | テストスタッフから依頼された補充内容が一覧に表示される | ☐ |

|---|---|---|
| オーナートークン URL を開く | テストオーナー用ページが表示される | ☐ |
| 清掃履歴に今回のテスト依頼が「確認済み」状態で表示される | 履歴が見える | ☐ |
| 履歴を開く → チェックリスト・写真・コメントが閲覧できる | 内容が見える | ☐ |

## 5. シナリオ E: 通知の到着確認

シナリオ A〜D 実施中に下記が届いていることを確認してください。届かない場合は迷惑メールフォルダや LINE の通知設定もご確認ください。

| 通知 | 受信者 | 媒体 | タイミング | 受信 |
|---|---|---|---|---|
| 新しい清掃依頼があります | テストスタッフ | LINE | A-6 で依頼作成した直後 | ☐ |
| 完了報告が提出されました | 管理者 | メール | B-4 で提出した直後 | ☐ |
| 清掃が完了しました | テストオーナー | メール | C で確認済みにした直後 | ☐ |
| 備品補充の依頼があります | 管理者 | メール | B-5 で依頼した直後 | ☐ |

> LINE 通知が届かない場合、テストスタッフが「LINE 公式アカウントを友だち追加していない」「LINE user ID が正しく登録されていない」のどちらかが原因の可能性が高いです。

---

## 6. 検収判定

すべてのシナリオで ✓ が付いたら検収完了です。下記のいずれかを工藤陸へご連絡ください。

- **✓ 検収完了** — 全シナリオ問題なく動作
- **△ 一部修正希望** — 動くが直してほしい箇所あり（箇所と希望内容を明記）
- **× 動作不良** — エラー / 想定外動作（再現手順・スクリーンショット添付推奨）

連絡先: 工藤陸（off.me.ton@gmail.com / LINE 個別）

---

## 7. 検収後の流れ

- 検収完了後、約 2 週間のテスト運用期間に入ります
- テスト運用中の不具合・改善要望は適宜対応
- 本格運用開始のタイミングは別途打ち合わせ
- 将来のアカウント移管（クライアント名義への切替）希望時はご相談ください
