# 民泊清掃管理アプリ 検収シナリオ

このドキュメントは、アプリが期待通り動くかを確認していただくための **検収用テストシナリオ** です。順番に試して、各ステップで「期待結果」が表示されたら ✓ を付けてください。最後まで一通り通れば検収完了となります。

所要時間: 60〜90 分（通知の到着待ち含む）

---

## 0. 準備するもの

- **PC（管理者操作用）** — ブラウザは Google Chrome / Edge / Safari いずれか
- **スマートフォン**（テスト用スタッフ・オーナーになる人の端末）
  - LINE アプリがインストール済み
  - メールが受信できる
- **テスト用メールアドレス 3 つ**（管理者・テストスタッフ・テストオーナー用。同じアドレスを使い回しても OK）
- **本アプリ URL**: `https://minpaku-cleaning.vercel.app`
- **管理者ログイン情報**（工藤陸から別経路で受領）
- **LINE 公式アカウントの友だち追加 QR / URL**（クライアント開設後、工藤陸から共有）

---

## 1. シナリオ A: 管理者として一周（PC で操作）

### A-1. ログイン

| 手順 | 期待結果 | ✓ |
|---|---|---|
| `https://minpaku-cleaning.vercel.app/admin/login` を開く | ログイン画面が表示される | ☐ |
| 受領した管理者メール・パスワードを入力して「ログイン」 | `/admin` ダッシュボードに遷移する | ☐ |

### A-2. オーナーを登録（先に登録が必要）

物件はオーナーに紐づくため、先にオーナーを登録します。

| 手順 | 期待結果 | ✓ |
|---|---|---|
| 左メニュー「オーナー」を開く | オーナー管理画面が表示 | ☐ |
| ページ上部のフォームに 名前「テストオーナー」、メール（テストオーナー用）を入力して「オーナーを追加」 | 下のリストに「テストオーナー」が追加表示 | ☐ |

### A-3. 物件を登録

| 手順 | 期待結果 | ✓ |
|---|---|---|
| 左メニュー「物件」を開く | 物件管理画面が表示。上部に物件登録フォーム | ☐ |
| オーナー選択で「テストオーナー」を選び、物件名「テスト物件」、住所「東京都〇〇区テスト1-2-3」を入力して「物件を追加」 | 下のリストに「テスト物件 — 東京都〇〇区テスト1-2-3」が追加表示 | ☐ |
| 追加された物件の **物件名「テスト物件」をクリック** | `/admin/properties/[id]` 物件詳細ページが開き、基本情報フォーム、担当スタッフ一覧（未割当表示）、オーナー用URLセクションが見える | ☐ |
| 物件詳細ページで「アクセス情報・備考」に「鍵は玄関ポスト」と入力し「保存」 | 保存成功（ボタンに「保存中...」が一瞬出て元に戻る）| ☐ |
| ページ右上「← 物件一覧へ」で戻る | 一覧画面に戻る | ☐ |

### A-4. スタッフを登録

| 手順 | 期待結果 | ✓ |
|---|---|---|
| 左メニュー「スタッフ」を開く | スタッフ管理画面が表示 | ☐ |
| ページ上部のフォームに 名前「テストスタッフ」、メール（テストスタッフ用）を入力。**「担当物件」セクションで「テスト物件」にチェック**を入れて「スタッフを追加」 | 下のリストに「テストスタッフ — テスト物件」と表示される（チェックを入れ忘れた場合は「担当物件なし」と出るので、その場合は次の手順で修正）| ☐ |

### A-5. スタッフ詳細画面で LINE ID を後から登録（テスト）

LINE 通知を受け取るには、スタッフが LINE 公式アカウントを友だち追加した後の `userId`（U で始まる文字列）を登録する必要があります。

| 手順 | 期待結果 | ✓ |
|---|---|---|
| スタッフ管理画面で「テストスタッフ」をクリック | `/admin/staff/[id]` スタッフ詳細ページが開く。基本情報フォーム、スタッフ用URLセクションが見える | ☐ |
| LINEユーザーID欄に **テスト用に何か文字（例: `Utest123`）** を入力して「保存」 | 保存成功 | ☐ |
| 「スタッフ用URL」セクションの「発行」ボタンを押す | URLが表示される。`https://minpaku-cleaning.vercel.app/staff/...` 形式 | ☐ |
| URL をコピーしてスマホに送る or QR 化して読み取る | スマホで開ける | ☐ |

> 実運用時の LINE ID 取得方法は `01-admin-guide.md` の §10「通知について」を参照。

### A-6. 物件にオーナー用URLを発行

| 手順 | 期待結果 | ✓ |
|---|---|---|
| 物件詳細ページ（A-3 で開いたページ）に戻る | 一覧から「テスト物件」をクリック | ☐ |
| 「オーナー用URL」セクションの「発行」ボタンを押す | URLが表示される。`https://minpaku-cleaning.vercel.app/property/...` 形式 | ☐ |
| URL をコピーしてスマホ or 別タブで開ける | オーナー用画面が表示される | ☐ |

### A-7. 依頼を作成

| 手順 | 期待結果 | ✓ |
|---|---|---|
| 左メニュー「依頼」を開く | 依頼管理画面が表示 | ☐ |
| 物件「テスト物件」を選び、チェックイン日に **明日以降の日付**、チェックアウト日に **その翌日以降**、人数「2」、備考「検収テスト用」を入力 | フォームに入力できる | ☐ |
| **試しに当日の日付**を選んで「依頼を作成」してみる | 「チェックイン日は翌日以降にしてください」とエラー表示 | ☐ |
| 正しい日付に直して「依頼を作成」 | 依頼一覧に「テスト物件」依頼が「未割当」で表示。テストスタッフ宛にメール届く（LINE は実トークン未設定時はメールに自動フォールバック） | ☐ |

### B-1. スタッフ URL を開く

| 手順 | 期待結果 | ✓ |
|---|---|---|

| 「提出する」をタップ | 「完了報告済み」状態に変わる | ☐ |

### B-5. 備品補充を依頼

| 手順 | 期待結果 | ✓ |
|---|---|---|
| 同画面下の「備品補充を依頼する」セクションに「テスト用トイレットペーパー × 1」を入力して送信 | 「依頼済み」表示 | ☐ |

## 5. シナリオ E: 通知の到着確認

シナリオ A〜D 実施中に下記が届いていることを確認してください。届かない場合は迷惑メールフォルダや LINE の通知設定もご確認ください。

| 通知 | 受信者 | 媒体 | タイミング | 受信 |
|---|---|---|---|---|
| 新しい清掃依頼があります | テストスタッフ | LINE → 失敗時メール | A-7 で依頼作成した直後 | ☐ |
| 完了報告が提出されました | 管理者 | メール | B-4 で提出した直後 | ☐ |
| 清掃が完了しました | テストオーナー | メール | C で確認済みにした直後 | ☐ |
| 備品補充の依頼があります | 管理者 | メール | B-5 で依頼した直後 | ☐ |

> LINE 通知が届かない場合、テストスタッフが「LINE 公式アカウントを友だち追加していない」「LINE user ID が正しく登録されていない」のどちらかが原因。その場合でも自動的にメールにフォールバック送信されます。

## 7. 検収判定

すべてのシナリオで ✓ が付いたら検収完了です。下記のいずれかを工藤陸へご連絡ください。

- **✓ 検収完了** — 全シナリオ問題なく動作
- **△ 一部修正希望** — 動くが直してほしい箇所あり（箇所と希望内容を明記）
- **× 動作不良** — エラー / 想定外動作（再現手順・スクリーンショット添付推奨）

連絡先: 工藤陸（off.me.ton@gmail.com / LINE 個別）

---

## 8. 検収後の流れ

- 検収完了後、約 2 週間のテスト運用期間に入ります
- テスト運用中の不具合・改善要望は適宜対応
- 本格運用開始のタイミングは別途打ち合わせ
- 将来のアカウント移管（クライアント名義への切替）希望時はご相談ください
