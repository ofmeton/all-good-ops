# RICE CREAM TOKYO — LINE公式 + ショップカード 設計書 v1

**作成日**: 2026-05-09
**対象**: RICE CREAM TOKYO（BEAT ICE 運営、新宿歌舞伎町）
**担当**: icecream-ops（業務委託先への提案責任）
**作業ディレクトリ**: `outputs/rice-cream-tokyo-line-shopcard/`

---

## 1. プロジェクト概要

LINE公式アカウントの「ショップカード機能」を活用し、近隣の日本人常連客への還元と緩やかなリピート促進を行う。**新規獲得マーケではなく、既存ファンへの感謝施策**として設計。インバウンド観光客向け施策とは分離する。

## 2. 目的

| 優先度 | 目的 |
|---|---|
| 一次 | いつも来てくれる近隣の日本人常連客への還元（感謝の表明） |
| 二次 | 関係性の継続によるリピート促進（金曜・寒い日のテコ入れ含む） |

## 3. 背景（資料根拠）

| 観点 | 内容 |
|---|---|
| 立地 | 新宿歌舞伎町、深緑ファサード、インバウンド一次・日本人二次（ブランドキット v1） |
| 客単価 | 約¥1,200（上位日も下位日も同等。客単価ではなく取引件数が売上ドライバー） |
| 構造的弱点① | 金曜が下位25%日の65%を占める（高売上日 vs 低売上日 多角的分析） |
| 構造的弱点② | 寒い日に売上が落ちる（上位16.3°C / 下位10.4°C） |
| ブランドトーン | 上品・米モチーフ・supermarket sale tags 不可（ブランドキット明記） |

近隣の日本人常連客は曜日・天候の弱日を補える唯一の顧客層。観光客は曜日・天候の影響を受けにくいため弱日テコ入れに寄与しない。

## 4. 対象顧客

近隣の日本人常連客（一般客）。具体的には:
- 歌舞伎町・新宿近辺で働く人
- 新宿近辺の住民
- 繰り返し足を運んでくれている層

※ BEAT ICE 関係者・スタッフ・オーナー知人は対象外（別途感謝の伝え方を持つ）。
※ インバウンド観光客は対象外（友だち追加 POP は日本語のみで誤誘導を防ぐ）。

## 5. 設計仕様

### 5.1 LINE公式アカウント

| 項目 | 内容 |
|---|---|
| アカウント名 | `RICE CREAM TOKYO` |
| ステータスメッセージ | `Japanese Rice Soft Serve / 関係者の方へ、季節の便りをお届けします。` |
| プラン | コミュニケーションプラン（無料・月200通まで）でスタート |
| あいさつメッセージ | 友だち追加時、感謝とショップカードの案内（テンプレ別途） |
| リッチメニュー | 初期は省略。友だち数が一定（目安 50名以上）になった段階で導入検討 |
| 自動応答 | 営業時間・アクセス FAQ のみ |

### 5.2 ショップカード機能

| 項目 | 値 |
|---|---|
| カード名 | `RICE CREAM TOKYO 関係者カード`（仮）|
| ポイント獲得方法 | スタッフが Official Account Manager で QR 表示 → 客がスキャン |
| 1会計あたりの加算 | 1pt |
| ポイント有効期限 | 6ヶ月（LINE 推奨。失効前リマインドは LINE 自動通知を有効化） |
| カードビジュアル背景 | Route C トーン（rice pulp #F3E8D6 + olive ink #384F35 + straw gold #B98B43 + 米モチーフ） |
| スタンプアイコン | 米粒 or ブランドエンブレム |

### 5.3 スタンプ・特典設計（軽め設定）

| ポイント数 | 特典 | 想定累計支払額（@¥1,200） | 還元率 |
|---|---|---|---|
| 3pt | トッピング 1種無料（チョコソース or 抹茶パウダー） | ¥3,600 | 約1〜1.5% |
| 7pt | ライスクリーム 1個無料 | ¥8,400 | 約4〜5% |
| 15pt | 関係者限定オリジナルグッズ（ステッカー or ミニトート 等） | ¥18,000 | 約3〜5% |

**設計意図**: 軽めの3/7/15で「達成感の頻度」を上げ、初期定着を強くする。3pt到達は約1ヶ月（月3回来店想定）で実現可能。

### 5.4 配信内容（不定期）

| カテゴリ | 頻度目安 | 内容例 |
|---|---|---|
| 季節の便り | 年3-4本 | 新茶の抹茶ラテ開始 / 秋の新米時期 / 冬季限定告知 / 棚田の近況 |
| 限定予告 | 年2-3本 | 新作試食 / 限定フレーバー登場 |
| 関係者限定告知 | 必要時（年1-2本） | 15pt到達特典の受け取り案内 / 弱日スポット案内（任意） |

**年間配信本数の上限**: 合計 6-9本（月1未満ペース）。これを超える場合は icecream-ops が事前に妥当性を確認する。ブロック率を抑えることを優先。

### 5.5 友だち追加の動線

| 配置 | 内容 |
|---|---|
| レジ脇 POP（A5）| 「いつもありがとうございます。関係者カードのご用意があります」+ QR コード |
| スタッフ声がけ | 押し売りせず、「LINE で関係者カードもありますよ」程度。希望者のみ |
| 既存 POP との関係 | Instagram POP / Google Maps POP / Student POP と並列。それぞれ役割を分ける |
| インバウンド対応 | 友だち追加 POP は日本語のみ。英語表記なし（誤誘導防止） |

## 6. 業務委託先（オーナー）への提案物

設計確定後、icecream-ops が以下を作成し、オーナーに提案・承認を得る:

1. **提案書（A4 1枚）** — 目的・施策概要・期待効果・初期コスト・運用負荷
2. **LINE公式アカウント運用イメージ** — モック画面 2-3（あいさつ・ショップカード表示・配信例）
3. **スタッフ向け運用マニュアル（1枚）** — QR 提示の流れ・声がけ例文・トラブル対応
4. **友だち追加 POP デザイン案** — Route C トーン準拠、A5 サイズ
5. **初期セットアップ手順書** — オーナーが Official Account Manager で行う作業ステップ
6. **関係者限定グッズの発注計画** — ステッカー or ミニトートの仕様・数量・発注先候補

## 7. 留意点・リスク

| カテゴリ | 内容 | 対処 |
|---|---|---|
| ブランドトーン整合 | Route C の手仕事感は紙カード手押しの方が強い。LINE 画面では rice pulp 紙質感や墨インクの陰影は再現しにくい | カード背景画像と配信文面で世界観を維持。妥協点として受容 |
| LINE 公式 仕様制約 | ショップカードのデザインは LINE UI 固定、背景画像とアイコン以外はカスタム不可 | 実装前に Official Account Manager の最新 UI を確認 |
| 年配層の友だち追加ハードル | 近隣常連客に高齢層が含まれる場合、QR スキャン操作で離脱の可能性 | スタッフが操作補助できるよう声がけスクリプトに含める |
| ポイント有効期限切れの不満 | 6ヶ月設定でも来店間隔が空く層から不満が出る可能性 | LINE 自動の失効前リマインドを有効化、配信文面で案内 |
| LINE 公式 ショップカード機能の仕様変更 | 2024年以降、仕様変更が時々ある | 実装フェーズで最新仕様を必ず確認、不一致あれば本書を改訂 |
| 配信頻度の上振れ | 月1超になるとブロック率上昇 | 配信前に必ず icecream-ops が「年間配信本数の枠内か」を確認 |

## 8. 期待効果（仮置き、検証指標は実装計画で確定）

- **一次目的**: 関係者層の感謝表明（定性。配信に対する反応・スタッフへの声で測る）
- **二次目的**:
  - LINE 友だち数（3ヶ月後 50-100名、6ヶ月後 100-200名 を目安として観測）
  - ショップカード保有者の月平均来店回数（保有/非保有比較）
  - 金曜日売上の前年同曜日比（弱日テコ入れ効果）

※ 数値目標は仮置き。実装計画フェーズでオーナーと合意の上、確定する。
