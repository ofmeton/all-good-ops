## 3. 自動化・効率化の余地

- **大型ファイル整理の初動テンプレ**: 「サイズ降順」「拡張子別件数」「同サイズ重複候補抽出」を1コマンドで出す。今回30+ターン使ったが、初動テンプレがあれば5-7ターン圧縮可能。
- **重複ファイル検出の3手順型**: 今回MD5照合を6回繰り返した（dmg・html・docx・jpg・mp4・LOGOネスト）。「同サイズ抽出 → MD5照合 → 一致判定」の3手順を型化。
- **macOS zip 展開の標準フロー**: `unzip -l` 確認 → 展開 → `__MACOSX/` 削除をテンプレ化。

## 4. 次回への改善提案

1. 整理タスクの初手は `du -sh */ | sort -hr` + 拡張子別件数 + 同サイズ重複候補抽出を1メッセージで実行（情報取得を分けない）
2. 整理方針合意の AskUserQuestion で「経理・会計系の独立扱い」「重複自動判定の許可」「中身確認の許可」を必ず含める
3. mv 前に必ず `ls -d <移動先>` で衝突確認（既存フォルダがあれば merge or リネーム判断）
4. macOS zip 展開時は `unzip -l zip | head` で root エントリ確認 → `-d` 先決定 → 展開後 `__MACOSX/` 削除を1セットで実行
5. 構造案を2回修正したら、構造ではなく前提（粒度・配置・例外ルール）が抜けているサインと捉え、前提質問に戻る
