## 1. 競合調査結果 (note 専用)

### 1.1 調査軸と手法

| 軸 | 手法 | コスト |
|---|---|---|
| 軸 1: 上位 X アカ 10 名の note 併用 | publishing research の posts JSON 全 10 アカで `note.com/` URL を grep | 0 円 (既存資産 reuse) |

### 1.2 主要 finding

#### finding 1: 上位 X アカは note 未運用 or X→note 送客なし

publishing research の 10 アカ tweets で `note.com/` URL 0 件。

→ **ofmeton は X 上位陣との直接競合が note には居ない** = 競合手薄。先行者利得を取りに行ける枠。

#### finding 2: note 内 AI 業務自動化ジャンルの上位作家マップ

ofmeton ターゲット (非エンジニア経営者) ど真ん中の作家:

| 作家 | URL | 特徴 |

| **KEN ＠非エンジニアのAI自動化** | https://note.com/ken_ai_dev7799 | Cursor 実践記シリーズ、¥300-980、非エンジニア × 経営者 |
| **yano yuzo** | https://note.com/gay_badger8909 | GAS×AI 35 本 / 50代AI 6 本 / マガジン 5 種 (243 本)、65 歳移住エンジニア |
| **ritowa** | (404、handle 要再特定) | 税理士 × AI 自動化、コード書けない視点 |
| **安井大樹** | (要 search 再特定) | 司法書士 × AI 業務自動化、第 1 回シリーズ |
| **Makasel** | https://note.com/makasel_ai | 中小企業向け実践、シリーズ展開 |

#### finding 3: 価格構造

- **無料 8 : 有料 2** (note 全体傾向)
- 価格中央値 **¥980** (¥300-1,500 の範囲)
- ¥2,980+ は ポス鳥 (コミック・評論) 等の高単価ニッチ
- v9 想定 (500-980 円) は中央値と整合

#### finding 4: 更新頻度

- note 上位は **数日単位** (日次〜週次)
- v9 想定「無料 3-5 本/月 + 有料 1 本/月」は note 上位平均より低頻度
- Phase 1 は v9 想定で開始、PCR/CVR 観察後に増やすか判断

#### finding 5: ターゲット明示の強さ

- 全員が「自分はこういう人」を **プロフィールで明示**:
  - 「コード書けません。税理士の私」(ritowa)
  - 「島暮らしの自動化エンジニア、65 歳で移住」(yano yuzo)

## 2. 構成パターンライブラリ (5 系統)

competitor analysis から抽出した 5 構成パターン。Writer の構成パターンライブラリに直接採用。

### 2.1 まとめ型

**特徴**: 大量の事例 / ツール / 職種を一括網羅
**例**: 「全 30 職種の AI×GAS 業務自動化コンプリート図鑑」(yano yuzo) / 「副業アイデア 16 選」(AI副業のアイデア手帖)

**構成テンプレ**:
```
1. リード (なぜ今このまとめが必要か、3 行)
2. 目次 (1-30 項目、各 1 行サマリ)
3. 本体: 各項目を 200-400 字 × N
4. 全体まとめ (どこから始めるべきかの優先順位)
5. CTA (関連まとめへの誘導 / プロフ送客)
```

**文字数**: 5,000-10,000 字 (大型まとめ) / 2,000-3,000 字 (小型まとめ)
**価格**: 無料 (集客用) or ¥980 (情報量を価値化)
**ofmeton 採用想定**: 月 1 本ペース、Tier 1 (業種別 SOP) と組み合わせる

### 2.2 段階型 (ロードマップ)

**特徴**: 入門 → 中級 → 上級の段階的ハウツー
**例**: 「月 3,000 円・1.5 万円・3 万円の 1 人社長コスト別設計図 3 選」(まーりん) / 「実践ロードマップ」系

**構成テンプレ**:
```
1. リード (どこに到達する記事か、現在地と目的地)
2. Step 1 (入門、最小コスト・最小学習)
3. Step 2 (中級、効果が出始める)
4. Step 3 (上級、収益化 / 拡大)
5. 各 Step での具体ツール・コスト・所要時間
6. CTA (次の Step へ進むためのリソース)
```

**文字数**: 3,000-5,000 字
**価格**: ¥980 (体系的価値)
**ofmeton 採用想定**: 月 1 本ペース、AI 委託フロー (Tier 1 T1-2) に適合

### 2.3 ツール比較型

**特徴**: 2-N 個のツール / 手法を体系比較
**例**: 「Zapier vs Make 徹底比較 2026 年版」(AIの教科書) / 「ChatGPT vs Claude vs Gemini」

**構成テンプレ**:
```
1. リード (なぜこの比較が必要か、結論先出し可)
2. 比較軸の定義 (5-10 軸)
3. ツール A の概要 + 各軸の評価
4. ツール B の概要 + 各軸の評価
5. (比較対象が 3 つ以上なら C, D...)
6. シナリオ別おすすめ (中小企業なら / 個人なら / 大企業なら)
7. CTA (各ツールの公式リンク / 関連解説記事)
```

**文字数**: 3,000-6,000 字
**価格**: 無料 (集客強い) or ¥500 (軽量)
**ofmeton 採用想定**: 月 0-1 本、Claude vs ChatGPT 等で書く時

### 2.4 専門職 × AI 型

**特徴**: 特定の専門職 (士業 / 業種) の視点で AI を解説
**例**: 「税理士の私が AI 業務自動化に挑戦」(ritowa) / 「司法書士が AI を語る理由」(安井大樹)

**構成テンプレ**:
```
1. 自己紹介 (どの専門職か、なぜ AI に挑戦するか)
2. 専門職特有の課題 (一般人には見えない業務の non-obvious な詰まり)
3. AI で解決できる箇所 / できない箇所の切り分け
4. 実装した具体例 (1-3 件、Before-After 数値開示)
5. 失敗談 (期待したけどダメだった部分)
6. 同業者への提言
```

**文字数**: 3,000-5,000 字
**価格**: 無料 (初回シリーズ) or ¥980 (深掘り)
**ofmeton 採用想定**: シリーズで月 1-2 本、Tier 3 (T3-1 士業特化 / T3-4 教育) と直結

### 2.5 シリーズ実践記型

**特徴**: 1 つのプロジェクトを連続記事で開示 (「— 5」「— 6」のように番号)
**例**: 「見積・請求・メール対応をボタン 1 つにした AI 業務自動化術【Cursor 実践記】ー 5」(KEN)

**構成テンプレ**:
```
1. 前回までの振り返り (シリーズの位置付け、3-5 行)
2. 今回のテーマ (1 つの機能 / 1 つの詰まり)
3. 試したこと (Step by Step、コード or 画面)
4. うまくいった / いかなかった
5. 次回予告
6. CTA (シリーズマガジン購読 / 前回記事リンク)
```

**文字数**: 1,500-3,000 字 (連続性のため軽め)
**価格**: 個別 ¥300-500 or マガジン購読 (推奨)
**ofmeton 採用想定**: 民泊清掃 SaaS / terra-isshiki HP 制作の連続シリーズ化

## 3. タイトル付け方ライブラリ

competitor analysis から抽出した 4 要素を Writer プロンプトに固定要素として追加。

### 3.1 必須 4 要素

| 要素 | 役割 | 例 |
|---|---|---|
| **【】カッコ** | 冒頭で記事の type を明示 | 【完全保存版】【最新】【第 1 回】【保存版】【実践記】 |
| **数字明示** | 量 / 価格 / 効果を具体化 | 「30 職種」「16 選」「月 3,000 円」「5 時間短縮」 |
| **権威付け表現** | コンテンツの網羅性 / 完成度を主張 | 「コンプリート図鑑」「徹底比較」「完全保存版」「集大成」 |
| **読者層明示** | 「これは誰のための記事か」をタイトルで明示 | 「非エンジニア」「1 人社長」「コード書けません」「IT 知識ゼロ」 |

### 3.2 タイトルテンプレ集

| パターン | テンプレ | 例 |
|---|---|---|
| まとめ型 | `【完全保存版】<読者層>のための<対象>×AI <数字>選` | 【完全保存版】1 人社長のための業務自動化 30 選 |
| 段階型 | `<読者層>が<期間>で<到達点>を実現する<数字>ステップ` | 非エンジニア経営者が 3 ヶ月で月 5 時間短縮を実現する 5 ステップ |
| ツール比較型 | `<ツール A> vs <ツール B> 徹底比較【<年>年版】` | Claude vs ChatGPT 徹底比較【2026 年版】 |
| 専門職 × AI 型 | `<専門職>の私が<対象>に挑戦する話【第 N 回】` | 司法書士の私が AI 業務自動化に挑戦する話【第 1 回】 |

### 3.3 NG パターン (競合分析で見つけた弱いタイトル)

- 抽象的: 「AI で業務効率化」「これからは AI 時代」

- 数字なし: 「いくつかの方法」「色々な選択肢」
- 権威付けなし: 「AI で楽になる話」

## 5. ティーザー境界設計テンプレ

### 5.1 設計原則

note 有料記事の「無料部分」「有料部分」の境界は **CVR の最重要レバー**。競合分析の観察から:

> **「無料部分で問題提起と Why を完結させ、有料部分で結論と How を提供」**

これが平均的な設計。ofmeton の差別化として、**無料部分に「無料でも実装可能な軽量版」を入れる** ことで「無料でも価値ある」体験を作り、有料部分への期待値を上げる。

### 5.2 無料部分の構成 (約 800-1,500 字)

```
1. リード (100-200 字)
   - 誰のための記事か (読者層明示)
   - 何が解決するか (1 文で)

2. 問題提起 (400-600 字)
   - 多くの人が抱えている具体的な詰まり
   - 既存解決策の限界 (時間 / コスト / スキル)
   - "なぜ今これに着手すべきか" (緊急性)

3. 無料でも実装可能な軽量版 (200-400 字)
   - 5 分で試せる入門編
   - これ単体で「読んでよかった」と思わせる

4. 有料部分への期待値 (100-200 字)
   - 有料部分で「結論 + 再現手順 + 数値開示」が来ることを予告
   - 「無料で十分」と「有料で全部欲しい」の境界明示
```

### 5.3 有料部分の構成 (約 1,500-6,000 字)

```
1. 結論 (300-500 字)
   - 答えを最初に提示 (read more させる前提)
   - 具体的な数値 / 設定値

2. 再現手順 (1,000-3,000 字)
   - Step by Step
   - スクショ / コード / プロンプト全文
   - 各 Step での所要時間 / コスト

3. ハマりどころと回避策 (300-500 字)
   - 同じ手順で詰まる箇所と対処

4. 応用例 (200-500 字)
   - 別業種への transfer
   - 関連シリーズへの誘導

5. CTA (100-200 字)
   - 個別相談 / プロフ送客 / メンバーシップ
```

### 5.4 ティーザー境界の A/B テストロジック (Optimizer 改善対象)

```
評価指標:
  - 無料部分の URL クリック率 (= note 訪問者)
  - note 訪問 → 購入 CVR (有料部分の魅力度)
  - 購入後の "スキ" / 通読率 (推定)

A/B 対象:
  - 「無料軽量版を入れる / 入れない」の有無
  - 無料部分の文字数 (800 / 1,200 / 1,500)
  - 有料部分の予告フレーズ (5 パターン)
  - 価格 (500 / 980 / 1,480) を同テーマで分けて A/B

Optimizer: 月次でテーマ × 価格 × ティーザー の CVR を集計、最適組合せを Writer に反映
※ ティーザー境界の "型" 変更は §5.1 Auto + brownout (反映後 7 日異常検知で巻き戻し)
```

## 7. 投稿時間最適化 (X / IG との連動)

### 7.1 基本タイムテーブル (Phase 1)

```
朝 7:00  - X 短文 (失敗談先行型) + IG カルーセル
朝 8:00  - note 新規記事公開 (X / IG 朝投稿の余波で送客)
昼 12:00 - X 短文 (具体 ROI 開示)
夕 17:00 - X 短文 (note 送客リンク付き、URL 課金許容)
夜 21:00 - X スレッド (深掘り) or note 補足コメント
夜 22:00 - IG ストーリー / リール (補助)
```

note は **朝 8:00 公開で X / IG の朝投稿余波を吸収** するのが基本パターン。

### 7.2 媒体連動詳細 (集客導線 3 パターン × 投稿時間)

v9 §4.8 の集客導線 3 パターン (A プロフ常時 / B 送客 RT 派生 / C 末尾 CTA) と組み合わせ:

| 時間帯 | X 投稿 | 集客導線 | note との連動 |
|---|---|---|---|
| 朝 7:00 | 失敗談先行型 (短文) | A (プロフ常時) | プロフ経由で note 一覧へ |
| 朝 8:00 | (note 公開時間) | — | 公開のみ |
| 昼 12:00 | ROI 数値 Before-After | A (プロフ常時) | プロフ経由 |
| 夕 17:00 | note 送客ツイート (URL 付き) | B (送客 RT 派生) | 直リンク (URL 課金) |
| 夕 17:30 | 朝/昼 ツイート派生 + 引用 RT | B (派生) | 17:00 ツイートを再露出 |
| 夜 21:00 | スレッド or 末尾 CTA | C (末尾 CTA) | プロフ誘導 CTA |

→ URL 付き投稿は **夕 17:00 の 1 本/日** に絞り、その他は A/C で URL 課金回避。Optimizer が URL 付き比率を最適化。

## 8. SEO 整備

### 8.1 note 内検索への最適化

- **タイトル先頭 30 字に主要キーワード**を入れる (note 内検索結果のプレビュー範囲)
- 本文冒頭 200 字にもキーワード重複 (検索インデックス強化)
- ハッシュタグ (#Claude #業務自動化 #非エンジニア など) は記事末尾に 3-5 個

### 8.2 Google 検索への最適化

note 記事は Google 検索でも上位表示されやすい (高ドメインオーソリティ):

- **タイトル**: 「<具体キーワード> + <数字> + <記事 type>」(例: "Claude 業務自動化 30 選 完全保存版")
- **見出し階層**: h2 / h3 を適切に使い分け
- **画像 alt text**: AI 生成画像でも alt text に「Claude 設定画面」「Before-After グラフ」等の説明を入れる
- **関連記事リンク**: 各記事の末尾に同マガジンの関連記事 3-5 件をリンク (内部 SEO)

### 8.3 競合キーワード調査 (今後の Optimizer 改善対象)

- competitor タイトルから主要キーワード抽出 → Writer が「これは含めるべき」と判定
- Google 検索結果の上位 10 件のタイトルパターン分析 → タイトル付け方ライブラリに反映
- 月次で Optimizer Phase 2 (Opus) が SEO Brief を生成

## 10. 媒体連動詳細 (補足)

### 10.1 note → X / IG への逆流

note 記事公開後、X / IG で以下を行う:

- 公開直後: X 短文「【新着】note 公開: <タイトル>」+ 主要 takeaway 3 点 (URL 付き、夕方枠で)
- 1 日後: X スレッドで「<タイトル>」の補足や読者コメントへの返信 (URL なし、プロフ誘導 = 集客導線 A)
- 1 週間後: IG カルーセルで note 記事をビジュアル化 (図解 9 枚)
- 1 ヶ月後: Optimizer が PCR / 購入率を集計、上位記事は「再露出」シリーズで再投稿候補に

### 10.2 X 投稿の note 引用パターン (集客導線 B の詳細化)

```
朝 7:00  X: 失敗談 (URL なし)
夕 17:00 X: note 公開告知 (URL 付き、¥0.20/req)
夕 17:30 X: 17:00 ツイートを引用 RT + 補足コメント (URL なし、課金回避)
夜 21:00 X: 17:00 ツイートを引用 RT + 別角度コメント (URL なし)
翌朝 7:00 X: 「昨日の note への反応まとめ」 (URL なし、プロフ誘導)
```

→ **1 つの note 記事を 5 派生で再露出**。URL 付きは 1 本のみで残り 4 派生は URL なし → URL 課金最小化。Optimizer が引用 RT の cascade ratio を観測。

### 10.3 Instagram カルーセルからの note 送客

IG は X と違い URL クリック導線が弱いため:
- カルーセル 9 枚の末尾 (9 枚目) に「note で続きを読む → bio link」と明示
- IG bio に Linktree や直リンクで note プロフィールを配置
- IG ストーリーで「ハイライト」を作り、note 記事への送客導線を常設
