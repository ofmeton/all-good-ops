#### avg❤ から見る fmat 効率の観察

- **短文 + 数字インパクト**: 最高 (@Shimayus 7,460) — 11k フォロワー帯でも超効率
- **長文批評**: 強い (@umiyuki_ai 2,710) — 思考量 / 独自視点が読者を引き付ける
- **箇条書きスレッド + 公式リソース**: フォロワー多いがエンゲージメント率は低い (@SuguruKun_ai 105k / 493 ≒ 0.47%)
- **ニュース速報型**: フォロワー多いが avg❤ 低い (@masahirochaen 193k / 169 ≒ 0.09%) — 速報は新規性で取るが熱量低い

#### ofmeton 向け fmat 選択への示唆

- **短文 + 数字** を主軸にエンゲージメント効率を狙う
- **長文批評** は独自視点を出す月 1-2 本 (頻度高すぎると読み手疲労)
- **箇条書きスレッド** は note 送客の中盤動線として
- **ニュース速報** は ofmeton ターゲット (非エンジニア経営者) には合わない、避ける

### 1.4 集客導線 3 パターンの具体運用 (v9 §4.8 の時系列化)

v9 §4.8 で「集客導線 3 パターン (A プロフ常時 / B 送客 RT 派生 / C 末尾 CTA)」を Optimizer 改善対象に追加した。v9.2 で具体的な 1 日タイムテーブルを定義:

```
朝 7:00  X 投稿: 失敗談先行型 (短文、Hook=「実は私も」型)
         集客導線: A (プロフ常時 note リンク経由)
         URL 付き? No (URL 課金回避)

昼 12:00 X 投稿: ROI Before-After 数字 (短文、Hook=「Before-After 数字」型)
         集客導線: A
         URL 付き? No

夕 17:00 X 投稿: note 送客告知 (短文、本日 note 記事 + URL)
         集客導線: B (送客ツイート + 引用派生)
         URL 付き? Yes (¥30/req コスト許容)

夕 17:30 X 投稿: 17:00 ツイート + 補足コメントで引用 RT
         集客導線: B 派生
         URL 付き? No

夜 21:00 X 投稿: 17:00 ツイート + 別角度コメントで再引用 RT
         集客導線: C (末尾「→ プロフィール参照」CTA)
         URL 付き? No

(or 21:00 を月 1-2 本だけ 長文単発 (思考フレーム) に置換)
```

→ **1 日 5 投稿のテンプレ固定**、URL 付き 1 本 (¥30 課金)、残り 4 本は URL なしで集客導線 A/B/C を使い分け。

月間 X コスト計算 (v9 §3.3 の更新):
- URL なし 4 × 30 = 120 投稿 × $0.015 = $1.80
- URL 付き 1 × 30 = 30 投稿 × $0.200 = $6.00
- self-watch 全 150 投稿 × 3 回 × $0.001 = $0.45
- 計 **$8.25 ≒ ¥1,287/月** (v9 §3.3 の ¥1,936 から下方修正可能)

### 1.5 投稿時間帯の最適化指針

publishing research に時系列観察は薄いが、一般的 Twitter 日本 ユーザーの active time から:
- 朝 7-9: 通勤読み (TL 流入のピーク)
- 昼 12-13: 昼休み (中規模ピーク)
- 夕 17-19: 退勤読み
- 夜 21-23: 自宅 active time (最大ピーク、保存・引用率高い)

ofmeton タイムテーブル (上記 §1.4) はこの time-of-day に整合。

Optimizer Phase 1 (Sonnet) が 月次でテーマ別 × 時間帯別の PCR を集計、±60 分の自動調整 (v9 §5.1 Full auto)。

### 1.6 X スレッド構成パターン (補強)

v9 §4.3 で「スレッド 2-10 本」とあるが、競合分析で見えた典型構造:

| パターン | 本数 | 構成 |
|---|---|---|
| 列挙型 (まとめ系) | 5-10 本 | Hook → 1 → 2 → ... → N → 結論 / CTA |
| 構造化解説型 | 3-5 本 | Hook → 背景 → 手順 → 注意点 → CTA |
| Q&A 型 | 3-4 本 | 質問 → 答え → 補足 → CTA |
| ストーリー型 | 4-7 本 | きっかけ → 試行錯誤 → 解決 → 学び → CTA |

ofmeton の中軸: **ストーリー型 (失敗談先行)** + **構造化解説型 (業種別 SOP)** の組み合わせ。

## 4. 横断改善観点 (X/IG/note 同時運用時)

### 4.1 1 トピックの 3 媒体展開フロー

```
1 つの核アイデア (Core Idea pool §3.1)
     │
     ├─→ X 短文 (朝・昼)
     │   失敗談 / ROI 数字
     │
     ├─→ X note 送客ツイート + 派生 (夕・夜)
     │   note タイトル + 主要 takeaway 3 点
     │
     ├─→ note 1 記事 (朝 8 時公開)
     │   構成パターン 5 系統から選択
     │
     ├─→ Instagram カルーセル 9 枚 (朝 9 時 / 夜 21 時)
     │   note 同内容を画像化、または独自視点
     │
     └─→ Instagram ストーリーズ
         カルーセル新着告知 + ハイライト追加
```

→ 1 core idea = 5-7 投稿の展開 = 月 30 トピック × 平均 5 投稿 = 月 150 投稿の運用ボリューム想定。

### 4.2 媒体間のカニバリ vs 補完

| 関係 | パターン | Optimizer 観察 |
|---|---|---|
| **補完** | X (短文 / 即時) → note (深掘り) → IG (保存型) | 健全な漏斗構造 |
| **カニバリ** | X 短文と Instagram カルーセル 1 枚目で全て言ってしまう (note 不要に) | 注意: note 訪問率低下 |
| **重複** | 同じ核アイデアで 3 媒体全部「同じ言い方」 | ブランド一貫性 OK だが新鮮さ無し |

Optimizer は **媒体間の遷移率 (X → プロフ → note)** を月次で観察、カニバリ兆候があれば Writer プロンプトに警告組込。

### 4.3 Optimizer の横断観察 (v9 §4.8 の補強)

v9 §4.8 で Optimizer Phase 1 は「(フォーマット, Hook 類型, 内容類型, 時間帯, 集客導線パターン) × (PCR, url_link_clicks, bookmark 率, インプ)」のクロス集計とした。v9.2 で追加:

- **媒体次元**: X / IG / note のクロスで「同じテーマがどの媒体で最も効率良いか」を観察
- **媒体間遷移率**: X PCR → プロフィール訪問 → note 訪問 → note 購入の漏斗で各段階の CVR を観察
- **横断補完スコア**: 3 媒体で同テーマを展開した時の合算エンゲージメント = 個別単独運用との差分 (補完効果の定量化)

### 4.4 1 トピックの 3 媒体テンプレ (集客導線統合)

```
core idea: 「Claude で Excel 集計を 5h → 15min に短縮」

X 朝 7:00:  失敗談「最初 Excel マクロで挑戦して挫折した」(短文、Hook=「実は私も」型)
            URL なし、集客導線 A

X 昼 12:00: 「Before 5h、After 15min。Claude 経由で実装した方法」(短文、Hook=「Before-After 数字」型)
            URL なし、集客導線 A

X 夕 17:00: 「今日の note 公開: Claude で Excel 5h → 15min【完全版】」(短文 + URL)
            URL 付き、集客導線 B

X 夕 17:30: 17:00 ツイート引用 + 「特に難しかったのは XX のところ」(派生)
            URL なし、集客導線 B
```
