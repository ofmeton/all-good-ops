- **改善履歴**: `data/improvement-log.jsonl` 4行。iteration 1〜4 すべて `status:"proposed"` のみで、`event:"cycle_complete"`・秘書審査結果（`secretary_decision`）行が一度も追記されていない。Phase 3（秘書審査・適用）が実質的に動いていないことを示唆。
- **proposal ファイル**: `outputs/improvements/` に `proposal-20260410.md` が **1つだけ** 存在。Iteration 3/4/5 がいずれも同日実行のため、`proposal-${DATE}.md` 命名により **同一ファイルを毎回上書き** している。Iteration 3 と 4 の提案実体は失われた。
- **discarded履歴**: ゼロ。重複制約は適用されない。

## 根本原因分析

1. **proposal ファイル命名の衝突** — `self-improve.sh:71` および :107 が `proposal-${DATE}.md` を使用。同日複数回実行（テストや手動再実行）で毎回上書きされ、iteration 単位の学習が不可能。Iteration 4 提案1で「永続化を担保」したが、永続化されたファイルが次イテレーションで自身に上書きされる構造的欠陥。
2. **Phase 3 が improvement-log に痕跡を残していない** — `self-improve.sh` には Phase 3（秘書審査）と Phase 4（cycle_complete メタ行追記）が組み込まれているにもかかわらず、`improvement-log.jsonl` には iteration 1〜4 の単純 `proposed` 行しかない。実際に走っている経路はこのスクリプトではなく、本プロンプトのみを単独実行する簡易テストハーネスと推定。Phase 3 が常時スキップ→改善が実適用されない→スコア空のまま、というループに陥っている。
3. **ベースライン採点が走らない** — Iteration 4 提案3 が SAFE/RISKY 判定すら受けていない。鶏卵問題は未解消で、毎イテレーションが「データなし」から開始されている。

総じて、ハーネスは **書き込みは増えたが、消化されない** 段階。次の最優先は学習の積み上げ構造（命名と履歴）を確立すること。

## 改善提案（優先度順、最大3件）

### 提案1: proposal ファイル命名に iteration 番号を含める [優先度1]

- **対象ファイル**: `scripts/self-improve.sh`
- **変更種別**: スクリプト修正
- **現状の問題**: `proposal-${DATE}.md` は同日複数イテレーションで衝突。学習履歴が常に1世代しか残らない。
- **具体的な変更内容**:
  - `scripts/self-improve.sh:11` 付近で `PROPOSAL_BASENAME="proposal-${DATE}-it${ITERATION}.md"` を定義
  - スクリプト内 3 箇所（プロンプト内パス、`PROPOSAL_FILE=` 行、Phase 3 プロンプト内パス、Phase 4 ログ記録）を `${PROPOSAL_BASENAME}` で統一
  - 互換のため、最新版を指す `outputs/improvements/proposal-latest.md` シンボリックリンクを `ln -sf "${PROPOSAL_BASENAME}" "${PROJECT_DIR}/outputs/improvements/proposal-latest.md"` で更新
- **期待するスコア影響**: improvement 軸 +5点相当（学習履歴の連続性確保）。次イテレーションから「前回提案 vs 今回提案」の差分検証が可能になる。
- **リスク**: 低。ファイル名変更のみ。既存 `proposal-20260410.md` は残る。

### 提案2: Phase 3 スキップを検知し improvement-log に明示する [優先度2]

- **対象ファイル**: `scripts/self-improve.sh`
- **変更種別**: スクリプト修正
- **現状の問題**: 簡易ハーネス（プロンプト単独実行）経由では Phase 3 が走らず、なぜスコアが伸びないかの可視性がない。`improvement-log.jsonl` を見ても "proposed" のままで、適用が止まっていることに気付きにくい。
- **具体的な変更内容**:
