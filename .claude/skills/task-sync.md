# タスク同期（Task Sync）

## 概要
各種ソース（メール、会議メモ、セッション中の発言）からタスクを抽出し、Asanaに同期する。重複チェックを含む。

## 実行タイミング
- morning-routine.sh で daily-scan → context-update の後
- セッション中に新しいタスクが発生した場合

---

## タスク抽出ルール

### 抽出トリガーキーワード
- 「〜する」「〜しなければ」「〜が必要」「〜を確認」「〜を送る」「〜を作る」
- 「TODO」「要対応」「締切」「期限」
- 「〜してください」「〜をお願い」（他者からの依頼）

### 抽出元
1. **メール**: daily-scanで取得した未読メールの中からタスクを抽出
2. **会話**: セッション中のユーザー発言からタスクを抽出
3. **会議メモ**: knowledge/context/ にある直近の更新からタスクを抽出

### タスクの属性
| 属性 | 必須/任意 | 説明 |
|------|----------|------|
| タイトル | 必須 | 動詞で始める |
| プロジェクト | 必須 | 4プロジェクトのいずれか |
| セクション | 必須 | 緊急度に応じて配置 |
| 期限 | 必須 | 不明な場合は仮期限 + 備考 |
| 説明 | 任意 | 背景情報・関連リンク |

---

## 重複チェックフロー

```
1. 新規タスク候補のタイトルから主要キーワードを抽出
2. asana_search_tasks でキーワード検索
3. 類似タスクが見つかった場合:
   a. 同一タスク → スキップ（ログに記録）
   b. 関連タスク → 既存タスクの説明に追記
   c. 別タスク → 新規作成
4. 判断に迷う場合 → 人間に確認
```

### 類似度の判断基準
- タイトルの主要キーワードが70%以上一致 → 同一の可能性高
- プロジェクト・セクションが同じ + キーワード50%一致 → 関連タスクの可能性
- それ以外 → 新規タスク

---

## セクション配置ルール

| 条件 | セクション |
|------|----------|
| 期限が今日 | 今日やること |
| 期限が今週中 | 今週中 |
| 期限が来週以降 + 定期確認要 | 継続フォロー |
| 条件待ち / 相手の返答待ち | 保留 |

---

## 同期結果の報告

```
📥 タスク同期結果

### 新規作成（X件）
- [プロジェクト] タスク名（期限: MM/DD）

### 更新（X件）
- [プロジェクト] タスク名 ← 情報追記

### スキップ（X件）
- タスク名（既存タスクと重複）

### 要確認（X件）
- タスク名（類似タスクあり、判断が必要）
```

## コスト管理
- 検索は1回のAPIコールで最大限カバー
- 大量のタスク候補がある場合はバッチ処理
- 目標トークン数: 1,500以内
