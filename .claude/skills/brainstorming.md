# ブレインストーミング（Brainstorming）

## 概要
HMW（How Might We）フレームワークを使い、発散→収束のプロセスで創造的なアイデアを生み出す。

## 利用エージェント
- strategic-advisor（メイン）
- ibasho-designer（居場所づくり関連）

---

## プロセス

### Phase 1: テーマ設定

ユーザーの課題をHMW形式に変換する。

**HMW変換ルール**:
- 「○○が困る」→「どうすれば○○を解決できるだろうか？」
- 「△△がしたい」→「どうすれば△△を実現できるだろうか？」
- 抽象的すぎる場合は分解して複数のHMWに

**例**:
- 「収入が不安」→「どうすれば月26万円を安定的に確保できるだろうか？」
- 「居場所を作りたい」→「どうすれば子どもが放課後に安心して過ごせる場所を作れるだろうか？」

---

### Phase 2: 発散（10分ルール）

**ルール**:
- 質より量（最低10個のアイデアを出す）
- 批判禁止（この段階では実現可能性を問わない）
- 自由奔放歓迎（突飛なアイデアも歓迎）
- 便乗OK（他のアイデアを組み合わせる）

**出力フォーマット**:
```
## 発散フェーズ: [HMW質問]

1. アイデアA — 一言説明
2. アイデアB — 一言説明
...
10. アイデアJ — 一言説明
（追加アイデアがあれば続ける）
```

---

### Phase 3: 収束

**評価基準**（各5点満点）:
| 基準 | 説明 |
|------|------|
| 影響度 | KGI/KPIへの貢献度 |
| 実現可能性 | 今のリソースで実行できるか |
| 緊急性 | 早く着手すべきか |
| ユニークさ | 差別化できるか |

**出力フォーマット**:
```
## 収束フェーズ

### 評価マトリクス
| アイデア | 影響度 | 実現可能性 | 緊急性 | ユニークさ | 合計 |
|---------|--------|-----------|--------|-----------|------|
| A | 4 | 5 | 3 | 2 | 14 |
| B | 5 | 3 | 4 | 4 | 16 |
...

### Top 3
1. **アイデアB**（16点）: 理由...
2. **アイデアA**（14点）: 理由...
3. **アイデアX**（13点）: 理由...

### 推奨する次のステップ
- Top 1について: ...
```

---

## 注意事項
- ブレストの結果はそのまま実行に移さない。推奨案は人間の承認を得てから具体化する
- アイデアの実行は熟議プロセス（deliberation）を経ることを推奨
- 発散フェーズでユーザーに「他に思いつくことはありますか？」と聞く（巻き込む）
