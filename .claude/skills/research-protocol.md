# リサーチ手順（Research Protocol）

## 概要
調査の手順、情報源の信頼性評価、結果の構造化ルール。

## 対象
- researcher（メイン）
- brand-publisher（市場調査）
- freelance-scout（案件リサーチ）
- admin-handler（制度調査）

---

## 調査プロセス

### Step 1: 調査設計
調査を始める前に以下を明確にする:

```
### 調査設計
- 調査テーマ: [何を調べるか]
- 調査目的: [なぜ調べるか / 何の判断に使うか]
- 必要な情報: [具体的にどんな情報が欲しいか]
- 情報の鮮度: [いつ時点の情報が必要か]
- 深さ: [概要レベル / 詳細レベル / 網羅的]
```

### Step 2: 情報収集
**優先順位**:
1. 公式サイト・一次情報（政府、企業公式、論文）
2. 専門メディア・業界サイト
3. 信頼性の高いニュースサイト
4. 個人ブログ・SNS（参考程度）

**ツール**:
- WebSearch: 広く情報を探す
- WebFetch: 特定URLの内容を取得
- Claude in Chrome: 動的サイトの情報取得

### Step 3: 信頼性評価

| ランク | 基準 | 例 |
|--------|------|-----|
| **高** | 公式・一次情報。更新日が明記。著者が明確 | 政府サイト、企業IR、学術論文 |
| **中** | 専門メディア。情報源が引用されている | 業界メディア、信頼性のあるニュースサイト |
| **低** | 個人見解。情報源不明。古い情報 | 個人ブログ、匿名掲示板、古い記事 |

### Step 4: 構造化・報告

## 調査結果フォーマット

```
## 調査レポート: [テーマ]

調査日: YYYY-MM-DD
調査目的: [目的]

### 要約
[3-5文で結論を先に述べる]

### 詳細

#### [サブトピック1]
- [事実1]（出典: [URL/サイト名], 信頼性: 高/中/低）
- [事実2]（出典: ..., 信頼性: ...）

#### [サブトピック2]
...

### 比較表（該当する場合）
| 項目 | 選択肢A | 選択肢B | 選択肢C |
|------|---------|---------|---------|
| ... | ... | ... | ... |

### 未確認事項
- [まだ確認できていないこと]

### 推奨アクション
- [調査結果に基づく推奨]

### 情報源一覧
1. [サイト名] — [URL]（信頼性: 高, 取得日: YYYY-MM-DD）
2. ...
```

---

## 注意事項
- 「調べましたが見つかりませんでした」も重要な結果。明記する
- 古い情報には「（YYYY年時点の情報。最新状況は要確認）」と付記
- 調査範囲が広がりすぎる場合は、最初にユーザーに確認して範囲を絞る
- コスト意識: 概要レベルで十分なら深掘りしない
