# 熟議プロセス（Deliberation）

## 概要
抽象的な課題、複数の正解がありうる課題、将来影響の大きい課題に対して、最低3回の会議を行い質の高い意思決定を行うプロセス。

## 発動条件
以下のいずれかに該当する場合:
- 戦略的意思決定（収入戦略、キャリア方針、長期目標の変更）
- 複数領域にまたがる判断
- 正解が一つに定まらない問題
- 将来への影響が大きい決定

## 発動前の確認
**熟議プロセスの開始には必ず人間の承認が必要。**
```
🔄 熟議プロセスに入ります

【テーマ】○○について
【理由】△△のため、3回会議で検討する必要があります
【推定トークン】約20,000トークン（推定コスト: XX円）
【所要時間】3ターン

→ 熟議プロセスを開始してよいですか？
```

## 月間制限
- **月2-3回まで**（コスト制約による）
- 超過する場合は人間に相談して優先順位を決定

---

## 3回会議プロセス

### 第1回: 課題定義会議

**目的**: 問題の構造を正確に把握する

**出力フォーマット**:
```
## 課題定義

### 目的
- この意思決定で何を達成したいか

### 制約条件
- 予算:
- 期限:
- リソース:
- 法令・規制:
- その他:

### 成功条件
- 何が実現されれば「うまくいった」と言えるか

### 不確実性
- 現時点で分からないこと
- 前提として置いていること

### 初期仮説
- 現時点での最善策の仮説（検証対象）

### ステークホルダー
- この決定に影響を受ける人・関係者
```

---

### 第2回: 選択肢比較会議

**目的**: 複数の選択肢を公平に評価する

**出力フォーマット**:
```
## 選択肢比較

### 選択肢一覧
| # | 選択肢 | 概要 |
|---|--------|------|
| A | ... | ... |
| B | ... | ... |
| C | ... | ... |

### 評価基準と重み
| 基準 | 重み | 説明 |
|------|------|------|
| 成功条件への寄与 | 30% | ... |
| コスト・リソース | 20% | ... |
| リスク | 20% | ... |
| 実現可能性 | 15% | ... |
| 感情・関係性への配慮 | 15% | ... |

### 各選択肢の評価
（選択肢ごとに評価基準別のスコアと根拠）

### 法令遵守チェック
- 各選択肢の法令・規制上のリスク

### 推奨
- 推奨案とその理由
- 次善の案
```

---

### 第3回: 実行計画会議

**目的**: 採択した案を実行可能な計画に落とし込む

**出力フォーマット**:
```
## 実行計画

### 採択案
- 選択肢X: （理由を簡潔に）

### 実行手順
| # | アクション | 担当 | 期限 | 完了条件 |
|---|-----------|------|------|---------|
| 1 | ... | ... | ... | ... |
| 2 | ... | ... | ... | ... |

### 計測方法
- 何を、いつ、どう測って成否を判断するか

### リスクと対策
| リスク | 確率 | 影響 | 対策 |
|--------|------|------|------|
| ... | ... | ... | ... |

### 見直し条件
- どのような状況になったら計画を見直すか

### 次のアクション
- 最初にやるべきこと（具体的に）
```

---

## 会議間の人間確認ポイント
- 第1回終了後: 「課題定義に漏れはないですか？」
- 第2回終了後: 「この評価で進めてよいですか？」
- 第3回終了後: 「この計画で実行してよいですか？」
