# Asana管理（Asana Management）

## 概要
秘書がAsanaのプロジェクト・セクション・タスクを管理するためのルール。タスク管理の中核システム。

## Asana プロジェクト構成

| プロジェクト | セクション | 用途 |
|---|---|---|
| **all-good-ops**（メイン） | 今日やること / 今週中 / 継続フォロー / 保留 | 日常タスク管理 |
| **finance** | 請求書 / 確定申告 / 月次〆 / 経費 | 経理・財務タスク |
| **kodomo-ibasho** | 居場所づくり / 社団法人設立 / リサーチ | ミッション系タスク |
| **business-ops** | Shopify / RICE CREAM / フリーランス案件 / 発信・ブランド | 事業運営タスク |

## タスク管理ルール

### タスク作成
- タイトルは動詞で始める（「確認する」「作成する」「連絡する」）
- 期限を必ず設定する（不明な場合は仮の期限を入れ、備考に「要確認」と記載）
- 適切なプロジェクト・セクションに配置

### セクション間移動
- **今日やること**: 当日中に完了すべきタスク
- **今週中**: 今週中に完了すべきタスク
- **継続フォロー**: 期限は先だが定期的に確認が必要
- **保留**: 条件が揃うまで待ちのタスク（保留理由を備考に記載）

### タスク完了
- 完了条件を満たしたタスクのみ完了にする
- 部分的に完了した場合は残作業を新しいタスクとして作成

### 重複チェック（task-sync時）
1. 新規タスク候補のタイトルで既存タスクを検索
2. 類似タスクが見つかった場合、統合するか新規作成するか判断
3. 判断に迷う場合は人間に確認

## 利用可能なMCPツール

### 参照系（自動実行可）
- `asana_get_tasks` — タスク一覧取得
- `asana_get_project_sections` — セクション一覧取得
- `asana_get_projects_for_workspace` — プロジェクト一覧取得
- `asana_search_tasks` — タスク検索
- `asana_list_workspaces` — ワークスペース一覧

### 変更系（秘書の判断で実行可、重要な変更は人間確認）
- `asana_create_task` — タスク作成
- `asana_update_task` — タスク更新
- `asana_create_project` — プロジェクト作成（人間確認必要）
- `asana_delete_task` — タスク削除（人間確認必要）

## 朝のルーティンでのAsana操作

1. 全プロジェクトの「今日やること」セクションのタスクを取得
2. 期限切れのタスクを検出して警告
3. 新しいタスクがあれば適切なセクションに配置
4. ユーザーに「今日やること」リストを提示

## 週次レビューでのAsana操作

1. 今週完了したタスクの一覧
2. 来週に持ち越すタスクの確認
3. 「保留」セクションの棚卸し
4. 新しいタスクの追加提案
