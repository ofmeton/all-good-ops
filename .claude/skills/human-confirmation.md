# 人間確認ルール（Human Confirmation）

## 概要
全エージェントが参照する必須スキル。人間の確認が必要な操作と、確認不要の操作を判定するためのルール。

## 基本原則
**「取り返しのつかない操作」「他者に影響する操作」「金銭が動く操作」は必ず人間に確認する。**

---

## 確認が必要な操作（MUST CONFIRM）

### 金銭関連
- 支払い・送金の実行
- 契約の締結・変更・解除
- 請求書の送付
- 価格の設定・変更
- 有料コンテンツの購入

### 外部送信
- メール（Gmail）の送信
- LINEメッセージの送信
- SNS投稿の公開
- ブログ記事の公開
- Slackメッセージの送信（社外向け）

### 法的手続き
- 届出書類の提出
- 法人登記関連の手続き
- 確定申告・税務申告
- 各種申請書の提出

### システム変更
- エージェント定義の追加・削除・大幅変更
- CLAUDE.md の変更
- .claude/settings.json の変更
- LaunchAgent の登録・変更・削除
- 本番環境に影響するスクリプトの実行

### 戦略変更
- 長期目標・KGI/KPIの変更
- 熟議プロセスの開始
- 収入戦略の変更

### 繊細な対人コミュニケーション
- 断りの連絡
- 条件交渉
- クレーム対応
- 相手の期待値を調整する連絡

## 確認不要の操作（AUTO-APPROVED）

### 情報収集・読み取り
- ファイルの読み取り
- WebSearch / WebFetch
- Gmail / Google Calendar / Asana の参照
- knowledge/ の参照

### 内部データの更新
- knowledge/context/ への追記・更新
- data/usage-log.jsonl への追記
- outputs/ への新規ファイル作成
- data/quality-scores.json の更新（月次監査時）

### 分析・計算
- データの集計・分析
- コスト推計
- スコアリング

## 確認の実施方法

### 確認時の提示フォーマット
```
⚠️ 人間確認が必要です

【操作】何をするか
【対象】誰に / 何に対して
【内容】具体的な内容（金額、文面、書類名など）
【影響】この操作の影響範囲
【取消可能性】取り消せるか / 取り消せないか

→ 実行してよろしいですか？（はい / いいえ / 修正して再提示）
```

### 判断に迷ったら
- **迷ったら確認する**。確認のコストは低い。取り返しのつかないミスのコストは高い。
- 「これは確認不要かもしれないが念のため」と付記して確認するのは問題ない。

---

## カテゴリ別の確認レベル

| レベル | 説明 | 例 |
|--------|------|-----|
| **BLOCK** | 確認なしでは絶対に実行しない | 送金、契約、法的提出 |
| **CONFIRM** | 確認を求め、承認後に実行 | メール送信、SNS投稿、予定作成 |
| **NOTIFY** | 実行するが事後報告する | context更新、usage-log追記 |
| **SILENT** | 報告不要 | ファイル読み取り、検索 |
