# コスト管理（Cost Control）

## 概要
月5,000円以内のAIコスト制約を維持するためのルール。秘書が主に参照するが、全エージェントがコスト意識を持つ。

## 月間コスト予算
- **上限**: 5,000円/月
- **日次目安**: 約166円/日
- **週次目安**: 約1,150円/週

---

## コスト分類

### 軽量（~2,000トークン）
- **基準**: 事実確認、簡単な計算、テンプレ出力、リマインド、スケジュール確認
- **処理方法**: 秘書が直接処理。サブエージェント不起動
- **応答フォーマット**: 箇条書き500文字以内
- **目標比率**: 全処理の50%以上

### 標準（~8,000トークン）
- **基準**: 分析、文面作成、手順実行、データ処理、1つの専門知識が必要
- **処理方法**: エージェント1名起動。スキル1-2冊参照
- **応答フォーマット**: 構造化レポート
- **目標比率**: 全処理の40%程度

### 熟議（~20,000トークン）
- **基準**: 戦略的意思決定、複数領域にまたがる判断、長期計画の設計
- **処理方法**: 3回会議プロセス（deliberation準拠）。開始前に人間確認
- **応答フォーマット**: 詳細レポート
- **目標比率**: 全処理の10%以下（月2-3回に制限）

## コスト節約テクニック

### 1. INDEX先読みパターン
knowledge/ やデータファイルを読むとき、まず INDEX.md や先頭数行を読んで必要な部分だけ本文を展開する。全文を先に読まない。

### 2. スキル参照の最小化
- **必須スキル**: 必ず読む
- **参考スキル**: 必要な場合のみ読む。「念のため」で読まない

### 3. 秘書の自己処理最大化
以下は秘書が直接処理し、サブエージェントを起動しない:
- 「今日の予定は？」→ Calendar MCP で直接取得
- 「タスクの状況は？」→ Asana MCP で直接取得
- 「メールある？」→ Gmail MCP で直接取得
- 簡単な計算、日付計算
- テンプレートベースの出力

### 4. 自動実行の制限
- `morning-routine.sh`: `--max-turns 20`
- `weekly-review.sh`: `--max-turns 15`
- `monthly-audit.sh`: `--max-turns 25`

### 5. 不要な会話ターンの削減
- 確認が明らかに不要な情報は一度に提示
- 「他に何かありますか？」を毎回聞かない

### 6. 外部 API / LLM 呼び出し新機能の実装前ユニットコスト試算（必須）
外部 LLM（Claude / Gemini / OpenAI / Codex CLI）や有料 API を**ユーザー操作ごと**に呼ぶ新機能を実装する際は、着手前に必ず以下を提示してユーザー確認を取る:

- 想定 input / output トークン数（1アクション）
- 該当モデルの単価（$/1M tokens を円換算）
- **1回あたりの想定コスト（円）**
- 想定利用頻度 × 月間想定額

**なぜ月額試算だけでは足りないか**: 「月額 5,000円以内」の枠内でも、1回あたりが数十円のアクションは心理的コストが高く「使わない」判断に至りやすい。個人事業の現場感として、**1回押すたびに数十円** はダッシュボード UI から連打する機能とは相性が悪い。

**運用ルール**:
- **1 アクション 10 円超**になる機能は必ず事前承認
- オーダーがつかない場合は「1 回空打ちして実測 → 提示」手順を取る（Codex の `--json` 出力等で tokens を拾って計算可能）
- 実装計画書・プロポーザルの段階で「1アクション単価」列を含める
- モデル単価は定期変動するので、Anthropic / Gemini / OpenAI の料金ページ直近値を参照

**事例**: 2026-04-23 ai-radar Day 5 で Codex CLI (`gpt-5-codex`) 深掘り機能を実装完了後「数十円/回なら使わない」と判明。476 行のコードがデッドコード化した。事前にこの節の試算を出していれば回避できた。

## コスト監視

### usage-log.jsonl の記録フォーマット
```json
{
  "timestamp": "2026-04-05T08:00:00+09:00",
  "session_type": "morning-routine",
  "cost_tier": "lightweight",
  "agents_invoked": ["secretary"],
  "skills_referenced": ["daily-scan"],
  "estimated_tokens": 1500,
  "summary": "朝のルーティン実行。3件のメール、2件の予定を報告"
}
```

### 月次コストチェック（usage-analyst が担当）
1. usage-log.jsonl を集計
2. コスト分類別の比率を確認（軽量50%以上か？）
3. 月5,000円ペースとの比較
4. 異常値の検出
5. 改善提案

### コスト超過時の対策（段階的）
1. **警告**: 月半ばで2,500円超 → 秘書がユーザーに警告
2. **制限**: 3,500円超 → 熟議を停止、標準処理も必要最小限に
3. **緊急**: 4,500円超 → 自動実行を一時停止、手動実行のみ
