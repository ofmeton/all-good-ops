# RICE CREAM 営業オペレーションマニュアル 設計書

- 起案日: 2026-05-15
- 担当: 工藤陸（店舗マネージャー） × Claude（構造化・スライド生成）
- 関連: `outputs/clients/rice-cream/` 配下、`memory/project_rice_cream_shop.md`

## 1. 目的

工藤陸がワンオペでやっている1日のオペレーションを、オコメン（アルバイト）が
**開店準備〜営業〜締めまで一人で回せる**ように、視認性最優先のスマホ縦長スライド
マニュアルとして整備する。

新店舗での運用が始まったばかりで、ソフトクリームマシーン操作を含む実手順は
工藤の頭の中にしかない。マニュアル化することでワンオペ依存を解消する。

## 2. 想定読者と利用シーン

- **読者**: オコメン（営業中のアルバイトスタッフ。新規入店時に渡される想定）
- **デバイス**: スマホ（LINEアルバム or カメラロールに保存して閲覧）
- **シーン**: 現場で手元のスマホを見ながら作業。指差しでフリック操作。
- **重要度**: 視認性最優先（明るい屋外・店内薄暗い環境両対応）

## 4. アウトプット仕様

### 4.1 ファイル形式と書き出し場所

- 形式: PNG（透過なし）連番
- サイズ: 1080 × 1920 px（縦長 9:16）
- 命名: `<phase>-<連番3桁>.png`（例: `open-001.png`, `biz-001.png`）
- 書き出し先: `outputs/clients/rice-cream/manual/`
- 締めフェーズは後日追加（本spec の範囲外）

### 4.2 スライドレイアウト（C案ハイブリッド）

```
┌─────────────────────────┐
│ ヘッダ帯（40px）        │  ← フェーズ名 / 連番   例: 開店準備  12/38
├─────────────────────────┤
│                         │
│                         │
│   写真領域（高さ70%）   │  ← トリミング済み写真
│   1280px相当            │
│                         │
│                         │
├─────────────────────────┤
│ 見出し（72-80px Bold）  │  ← ステップ短文
│ 本文（48-56px）         │  ← 説明文
│ ［赤帯：注意点］        │  ← 必要時のみ
│ ［青帯：補足］          │  ← 必要時のみ
└─────────────────────────┘
                 文字領域（高さ30%）
```

- 背景: 文字領域は白
- フォント: Noto Sans JP Bold（ブランドフォントは使わない）
- 配色: 白黒ベース＋赤（注意）＋青（補足）の3色のみ
- **ブランドキットは取り込まない**（視認性最優先のため）

### 4.3 写真の処理

- **トリミングのみ**（背景透過処理なし）
- PDFから対応ページを画像抽出 → 主題部分でクロップ → 4:3 or 16:9 に整える
- 「2階の荷物」の一覧スライドも背景はそのまま、各アイテムを四角形でクロップして並べる
- ぼけ・ブレが酷い写真は**追加撮影リスト**に回す

## 5. スライド構造化のルール

工藤の殴り書きを以下のパターンで分解する。

| 元メモのパターン | スライド化 |
|---|---|
| 文章 → 写真1枚 | 1スライド：写真+見出し+本文 |
| 文章 → 写真複数枚 | 写真ごとに連続スライド分割 |
| 写真なしの文章のみ | **テキストオンリースライド**（白背景に大文字、写真領域を文字に転用） |
| 「※追加検討中」「写真にはないですが」等 | 該当スライド末尾に赤帯（注意）or 青帯（補足）として載せる |
| 工藤の口語（｢たのむ｣｢〜です｣ など） | オコメン向けの**業務語**に書き直す |

### 5.1 フェーズ構成

- **Phase 1: 開店準備**（target: 30-45 スライド想定）
  - 到着 → 鍵 → 入店
  - アイスマシーン立ち上げ
  - 2階から荷物下ろし（一覧スライド含む）
  - セッティング（抹茶マシーン・コンセント・冷蔵庫投入・電飾）
  - 外セッティング（看板・テーブル）
  - 電気ポット使い方
  - 開店前最終チェック（要追加ヒアリング）

- **Phase 2: 営業中**（target: 8-15 スライド想定）
  - 会計編（スマッパチケット・インバウンドキー）
  - アイスマシーン補給ランプ点灯時対応

- **Phase 3: 締め**（本spec範囲外。次フェーズで追加ヒアリング）

## 6. 「to claude:」指示の個別処理

### 6.1 アイスマシーン停止→運転（開店時）

工藤指示：「取扱説明書で該当部分参照してうまいこと引用して。わかりやすくたのむ」

- 取説 NA-1412AE の **始業時操作ページ**を参照
- 3スライドに分解：
  1. 停止ボタンの位置と押し方
  2. 運転ボタンの位置と押し方
  3. 保冷ランプ点灯・出口アルコール消毒など正常起動の確認
- 取説原文は引用せず、**現場語で短く書き直す**

### 6.2 2階から下ろす荷物（一覧スライド）

工藤指示：「ものをくり抜いてひとまとめにパッと1ページにまとめて見させたい」

- **透過処理なし・トリミングのみ**でアイテム別に切り出し
- 1スライドに2×3 or 2×4 グリッドで配置
- 各画像下に名称ラベル（段ボール / レジ / 白ケース / 置物 / グッズ / 材料 / 洗い物 / テーブル&クロス / 看板 / ソフトミックス / ライスミルク / ロックアイス）
- アイテム数が6を超える場合は2スライドに分割

### 6.3 アイスマシーン補給ランプ点灯時（営業中）

工藤指示：「取扱説明書参考に頼んだ」

- 取説 NA-1412AE の **補給手順ページ**を参照
- 3スライドに分解：
  1. 補給ランプの見た目（写真）
  2. 補給するもの（ソフトミックス）と取り出し場所
  3. 補給後の確認（ランプ消灯）

## 7. 写真不足の扱い

PDFを全ページ確認した後、**追加撮影リスト**を工藤に返す。

- 例：鍵を回す瞬間 / 抹茶マシーン置き場の俯瞰 / 沸騰ボタン押下後のポット表示 / 注意したい配線位置 など
- 工藤がメモアプリに追加撮影を殴り書き → Claude が差し込み更新
- 不足写真がある場合は該当スライドを**テキストオンリー**で先に作っておき、後で写真差し込みで上書き

## 8. 作業フロー

1. **写真抽出**: 開店準備PDF 40p をページ画像化（pdftoppm）→ 写真をクロップ
2. **モック1枚作成**: テンプレートの妥当性確認のため、開店準備のうち1スライド（例: アイスマシーン停止ボタン）を完成形で生成 → 工藤レビュー
3. **モック承認後、全スライド一気に生成**（開店準備フェーズ）
4. 同じ要領で営業中フェーズ
5. **追加撮影リスト**を工藤に返す
6. 追加写真受領 → 差し込み更新
7. 締めフェーズの殴り書き依頼（次の spec として別扱い）

## 9. 完了条件

- 開店準備フェーズの全スライドが `outputs/clients/rice-cream/manual/open-NNN.png` として書き出されている
- 営業中フェーズの全スライドが `outputs/clients/rice-cream/manual/biz-NNN.png` として書き出されている
- 不足写真がある箇所はテキストオンリースライドで暫定埋め
- 追加撮影リストが `outputs/clients/rice-cream/manual/_TODO_photos.md` として残されている

## 10. 非対象 / YAGNI

- ブランドキット適用（色・フォント）→ 視認性優先のため使わない
- 写真の背景透過処理 → トリミングで十分
- 動画・GIF → 静止画スライドのみ
- PDF化 → LINEアルバム想定で個別画像のみ
- 締めフェーズ → 殴り書き未提出のため本spec範囲外
- 旧店舗（御徒町アメ横44）の情報 → 無視
- 多言語化 → 日本語のみ

## 11. 技術スタック

- 画像生成: Python + Pillow（テキスト合成）/ pdftoppm（PDF→PNG）
- venv: `~/.venvs/img-tools/`（既存）
- フォント: Noto Sans JP（システムまたは Google Fonts ダウンロード）
- 必要ライブラリ: `Pillow`, `pdf2image` または `pdftoppm` CLI

## 12. レビュー観点

- 視認性（テキストサイズ・コントラスト）が現場の明るさに耐えるか
- 1スライドに情報を詰めすぎていないか（オコメンが指差しで追えるか）
- 工藤の意図しない解釈で文を変えていないか
- 取説引用部分が正確か（NA-1412AE の操作と齟齬がないか）
